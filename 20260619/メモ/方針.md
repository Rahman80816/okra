# オクラ収穫タスク 方針

## 最終ゴール

ロボットがオクラを認識し、右手で収穫し、左手のかごに入れること。

## 今回のタスク

右手でどの位置でオクラを収穫しても、左手にあるかごに必ず入れることができるようにする。

## 自分の担当についての要件

- 右手がどの位置でオクラを収穫しても、左手のかごに確実に入れられること
- 収穫位置に依存しない汎用的な動作プログラムを作成する

## 技術調査

### 前提（現在のシステム構成）

- ロボット: Unitree G1 EDU Ultimate
- 右手: Dex1-1 グリッパー + 切断把持エンドエフェクタ
- 左手: かご固定（制御なし、kpホールドで姿勢維持）
- 右腕制御: `rt/arm_sdk`（DDS, 250Hz）で上半身17DoF（腕14+腰3）
- グリッパー制御: `rt/dex1/right/cmd`（DDS → dex1_1_service → RS-485）
- 操作ポリシー本線: ACT（模倣学習）で右腕7DoF + 右グリッパー1DoF = 8DoF
- 左腕は起動姿勢でkp保持されており、かごの位置は常に一定

### タスクの性質

「オクラを左手のかごに入れる」動作は以下の特徴を持つ:
- **開始位置は毎回異なる**（収穫位置に依存）
- **終了位置は毎回同じ**（左手のかご上空 → リリース）
- つまり「任意の開始位置 → 固定の終了位置」への移動問題

### アプローチ候補

#### アプローチA: 固定関節角度への移動（プログラマティック方式）

**概要**: 左手のかご上空に対応する右腕の「投入姿勢」の関節角度を事前に計測し、収穫後はその固定姿勢へ補間移動してリリースする。

**手順**:
1. テレオペまたは手動で右手をかご上空に配置し、そのときの右腕7関節の角度を `rt/lowstate` から読み取って記録
2. 収穫後、現在の関節角度から投入姿勢へ線形補間（または台形速度プロファイル）で移動
3. 到達したら右グリッパーを open してオクラをリリース

**メリット**:
- 実装がシンプル（Pythonで数十行）
- ACTの学習データ不要
- 動作が確実・再現性が高い
- すぐに動作確認できる

**デメリット**:
- 補間中の障害物回避がない
- 中間経路で自己干渉の可能性（要確認）
- かごの位置が変わった場合は再計測が必要

**使用技術**: unitree_sdk2py + DDS（rt/arm_sdk, rt/lowstate, rt/dex1/right/cmd）

#### アプローチB: ACT（模倣学習）に投入動作も含めて学習

**概要**: テレオペで「様々な位置で収穫 → かごに入れる」のデモを多数収集し、ACTに収穫からかご投入までの一連の動作を学習させる。

**メリット**:
- 収穫からかご投入まで一つのポリシーで一貫
- 中間経路も自然な動作になる

**デメリット**:
- 大量のデモデータ収集が必要
- 投入動作の汎化が保証されない
- 学習・デバッグに時間がかかる

### 結論・方針

**アプローチAで投入動作を実装する。** → 上田さん了承済み（2026-06-17）

理由:
1. 自分の担当範囲は「右手でとったオクラを左手のかごに入れる」部分のみ
2. 固定軌道方式であれば、ACTの学習データや認識システムを待たずに単独で開発・検証できる
3. 左手のかご位置は固定されているため、固定関節角度で十分対応可能

### 開発の進め方

**まずMuJoCoシミュレーション上で動作確認してから、実機に移行する。**（上田さんアドバイス）

```
ステップ1: MuJoCoシミュレーションで開発・検証
  → 実機なしでPC上で右腕の動きを目視確認できる
  → 自己干渉やおかしな軌道を事前に発見できる
  → 壊す心配がない

ステップ2: 実機で動作確認
  → シミュレーションで確認済みのプログラムを実機に適用
```

MuJoCo環境は既にこのPC上にある:

- MuJoCoワークスペース: `/home/techshare/drl_kit/mujoco_ws/ts_mujoco-main/`
- G1モデル（手付き）: `unitree_robots/g1/scene_29dof_with_hand.xml`
- Python制御スクリプト: `simulate_python/unitree_mujoco.py`
- DDS連携ブリッジ: `simulate_python/unitree_sdk2py_bridge.py`（シミュレーション上でもrt/arm_sdk等のDDS通信が使える）

### 実装に必要なもの

1. **rt/lowstate からデータ受信**（現在の関節角度を取得）
2. **rt/arm_sdk へデータ送信**（目標関節角度を送信、250Hz）
3. **rt/dex1/right/cmd へデータ送信**（グリッパー開閉）
4. **投入姿勢の関節角度の事前計測**（テレオペまたは手動で配置して記録）
5. **補間ロジック**（現在角度 → 目標角度を滑らかに補間）

### 関節構造の情報（URDF）

G1の関節間の距離や構造はURDFファイルに定義されている。

- ファイル: `/home/techshare/ILkit/xr_teleoperate/assets/g1/g1_body29_hand14.urdf`

右腕の関節チェーン（関節間の距離）:

```
torso（胴体）
  │ xyz = (0.004, -0.100, 0.248)  ← 胴体から右肩まで (m)
  ↓
右肩ピッチ (right_shoulder_pitch)
  │ xyz = (0, -0.038, -0.014)
  ↓
右肩ロール (right_shoulder_roll)
  │ xyz = (0, -0.006, -0.103)
  ↓
右肩ヨー (right_shoulder_yaw)
  │ xyz = (0.016, 0, -0.081)
  ↓
右肘 (right_elbow)
  │ xyz = (0.100, -0.002, -0.010)
  ↓
右手首ロール (right_wrist_roll)
  │ xyz = (0.038, 0, 0)
  ↓
右手首ピッチ (right_wrist_pitch)
  │ xyz = (0.046, 0, 0)
  ↓
右手首ヨー (right_wrist_yaw) → ここにグリッパーが付く
```

xyzの数値は「親の関節から子の関節までの距離（メートル）」。
