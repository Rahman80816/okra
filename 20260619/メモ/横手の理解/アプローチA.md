# アプローチA: 固定関節角度への移動

## ざっくり言うと

「右手を、決まったポーズに動かして、手を開く」だけ。

## ロボットの腕の基本

G1の右腕は**7つの関節（モーター）**でできている。肩3つ、肘1つ、手首3つ。各関節には**角度**があり、例えば：

```
肩ピッチ: 0.5 rad
肩ロール: -0.3 rad
肩ヨー:   0.1 rad
肘:       1.2 rad
手首ロール: 0.0 rad
手首ピッチ: -0.1 rad
手首ヨー:   0.3 rad
```

この7つの角度の組み合わせで、右手の位置と向きが決まる。

## やることの流れ

### ステップ1: 「投入姿勢」を事前に決めておく

人間が手動で右手をかご上空に持っていき、そのときの7つの関節角度を記録する。
これが「投入姿勢」（ゴール）。

### ステップ2: 収穫後、今の関節角度を読む

rt/lowstate（DDSトピック）からデータを受信すると、今の7関節の角度がリアルタイムでわかる。
これが「現在姿勢」（スタート）。

### ステップ3: 現在姿勢 → 投入姿勢へゆっくり動かす

急に動かすと危険なので、少しずつ目標を変えていく。
例えば250ステップ（1秒間）かけて:

```
各ステップの目標 = 現在角度 + (投入角度 - 現在角度) × (i / 250)
```

これをrt/arm_sdk（DDSトピック）に250Hzで送り続ける。

### ステップ4: 投入姿勢に着いたら手を開く

rt/dex1/right/cmd に「開く」指令を送る。
オクラがかごに落ちる。

## なぜ「どの位置でも」対応できるのか

ポイントは**ゴール（投入姿勢）が毎回同じ**ということ。

```
収穫位置A --> 投入姿勢 --> 手を開く
収穫位置B --> 投入姿勢 --> 手を開く
収穫位置C --> 投入姿勢 --> 手を開く
```

スタート位置が毎回違っても、「今の角度」と「目標の角度」の差を計算して補間するだけなので、自動的にどの位置からでも同じゴールに到達する。

## 必要な技術要素

| 要素 | 何をする | 難しさ |
|---|---|---|
| DDSの送受信 | 関節角度の読み書き | Pythonライブラリで対応可 |
| 投入姿勢の計測 | 1回手動で配置して記録 | 簡単 |
| 補間計算 | 線形補間（算数レベル） | 簡単 |
| グリッパー開閉 | DDSコマンド送信 | 石丸くんの成果を流用 |

機械学習もAIも不要で、純粋にプログラミングだけで実現できるアプローチ。
