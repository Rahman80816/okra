# アプローチB: ACT（模倣学習）に投入動作も含めて学習

## ざっくり言うと

「人間のお手本を見せて、ロボットに真似させる」方式。

## 流れ

1. 人間がVRゴーグルでロボットを遠隔操作して、「オクラを取る → かごに入れる」を何十回もやる
2. そのときの関節角度やカメラ映像を全部記録する → これが「お手本データ」
3. お手本データをAIに学習させる → AIが「こういう状況ではこう腕を動かせばいい」を覚える
4. 本番ではAIが自分で判断して腕を動かす

## アプローチAとの違い

```
■ アプローチA（固定軌道）
  人間が「ゴールの姿勢」を1つ決める
  → プログラムが今の位置からゴールまで直線的に動かす
  → 毎回同じ経路のパターンで動く

■ アプローチB（模倣学習）
  人間が「お手本の動作」を何十回も見せる
  → AIがお手本のパターンを学習する
  → 状況に応じてAIが動き方を判断する
```

## 具体的なイメージ

### お手本収集（テレオペレーション）

```
  ┌──────────┐        ┌──────────┐
  │ VRゴーグル  │ ─────→ │  G1ロボット │
  │（人間が操作）│  無線   │（人間の動きを再現）│
  └──────────┘        └──────────┘
                           │
                     関節角度・カメラ映像を記録
                           ↓
                      ┌──────────┐
                      │ お手本データ │
                      │ （50〜100回分）│
                      └──────────┘
```

### AI学習

```
  お手本データ → GPU（高性能PC）で学習 → 学習済みモデル（ACTポリシー）
```

### 本番

```
  カメラ映像 → 学習済みモデル → 「次はこう動かす」→ ロボットの腕
```

## メリット・デメリット

|  | アプローチA（固定軌道） | アプローチB（模倣学習） |
|---|---|---|
| 動きの自然さ | 機械的（直線移動） | 人間っぽい自然な動き |
| 準備の手間 | 姿勢を1回計測するだけ | お手本を何十回も操作 |
| 開発時間 | 数日 | 数週間（データ収集+学習） |
| 確実性 | 高い（毎回同じ動き） | 不確実（学習がうまくいくか次第） |
| 必要な設備 | PC + ロボットだけ | VRゴーグル + GPU + ロボット |

## 自分の担当との関係

今回の自分の担当は「かごに入れる」部分だけなので、アプローチBは少しオーバースペック。ACTは本来、収穫動作（オクラを見つけて掴む）のような複雑な動作に向いている。「決まった場所に運んで手を開く」だけならアプローチAで十分。
