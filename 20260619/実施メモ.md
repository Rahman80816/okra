## 2026-06-17 `arm_interpolator.py` 実装

作成ファイル:

- `src/arm_interpolator.py`

実装内容:

- `ArmInterpolator` クラスを実装した。
- `start_angles` と `goal_angles` は同じjoint集合であることを検証する。
- `duration_s` は非負、`control_freq_hz` は正の値であることを検証する。
- `total_steps = max(1, round(duration_s * control_freq_hz))` とし、ゼロ秒指定でも終点へ到達可能にした。
- `get_target(step)` は線形補間で目標角度dictを返す。
- `step <= 0` は始点、`step >= total_steps` は終点にクランプする。
- `is_done(step)` は `step >= total_steps` で完了判定する。

検証:

- `python3 -m py_compile src/arm_interpolator.py` 相当の文法チェックに成功。
- 簡易チェックで `duration_s=1.0`, `control_freq_hz=4` のとき `total_steps=4`、`step=2` で中間値、`step=4` で終点になることを確認した。

- MuJoCo制御は、各jointの現在角度・速度からPDトルクを計算し、actuator ctrlrangeでクリップして `data.ctrl[actuator_id]` に入れる。

## 2026-06-17 実機デプロイ方法の方針

実機デプロイの基本方針:

- MuJoCo用 `drop_to_basket_mujoco.py` をそのまま実機で動かさない。
- 実機用に `drop_to_basket_real.py` を別途作る。
- 共通利用するのは `arm_interpolator.py`。
- MuJoCo版で得た関節目標を、実機ではUnitree DDSの `LowCmd` で送る。
- 実機では `qpos` 直接代入やfloating base固定は使えない。
- 実機では必ず現在関節角を `LowState` から読み、そこから安全に補間する。

実機版スクリプトの処理案:

1. `ChannelFactoryInitialize(domain_id, "enp8s0")` でDDS初期化。
2. `LowState` subscriberを立て、現在の全関節角 `q` / 速度 `dq` を受信できるまで待つ。
3. G1の右腕7関節・左腕7関節・腰3関節のmotor index対応を定義する。
4. 現在姿勢を開始姿勢として読み取る。
5. 左腕・腰・脚など、動かさない関節は現在角度でホールドする。
6. 右腕だけを目標投入姿勢へ `ArmInterpolator` で補間する。
7. 各周期で `LowCmd.motor_cmd[i]` に `q`, `dq=0`, `kp`, `kd`, `tau=0` を入れて送る。
8. 到達後、右グリッパーを開く。
   - 手指がDDSで制御できるなら手指motorへ開き角度を送る。
   - 既存のDex gripper serverを使うなら、別プロセス/コマンドで開く。
9. 完了後も数秒ホールドし、安全に停止する。

安全上の注意:

- 最初は右腕だけ、小さい移動量、低い速度制限で確認する。
- 実機ではMuJoCoのIK結果を過信しない。
- 実機の現在姿勢から目標姿勢までの最大関節差分を見て、速度制限から移動時間を決める。
- `kp/kd` は低めから始める。
- いつでも停止できる人が近くにいる状態で実行する。
